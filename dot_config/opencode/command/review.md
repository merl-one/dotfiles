# Code Review

1. Execute `git diff` via terminal. 
2. Analyze all changed logic from multiple perspectives: Correctness, Security, Architecture, Performance.
3. Check for typical red flags, sloppy types, unhandled errors, or over-engineering.
4. Output a summary report categorized by severity (Critical, High, Medium, Low).
5. DO NOT edit the code.
